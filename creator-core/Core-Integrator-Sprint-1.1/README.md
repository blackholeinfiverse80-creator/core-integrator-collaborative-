# Core-Integrator-Sprint

Unified Backend Bridge - A FastAPI orchestration platform for managing multiple AI agents and modules.

## Creator Core Engine

This repository now includes a Creator Core engine that accepts deterministic Prompt Runner instruction JSON and converts it into Blueprint Envelopes for BHIV Core routing.

### Creator Core Endpoint

`POST /creator-core/generate-blueprint`

Example request:

```json
{
    "prompt": "Explain Newton's laws of motion with real-life examples for a high school student.",
    "module": "education",
    "intent": "explain_concept",
    "topic": "physics_newtons_laws",
    "tasks": [
        "define_first_law",
        "define_second_law",
        "define_third_law",
        "provide_real_life_examples"
    ],
    "output_format": "step_by_step_guide",
    "product_context": "creator_core"
}
```

The engine emits a deterministic blueprint envelope, persists an immutable blueprint artifact locally, and writes InsightFlow-compatible telemetry events to local JSONL storage.

## Recent changes (local-only blueprint workflow)

- Blueprints are generated by `POST /creator-core/generate-blueprint` and saved locally as immutable JSON artifacts under `db/creator_core_bucket`.
- By default the service does NOT forward generated blueprints to BHIV Core. This avoids accidental remote execution and keeps generated artifacts private to your repository.
- To enable forwarding to BHIV Core set the environment variable `ENABLE_BHIV_FORWARD=true` and restart the app. The BHIV endpoint used is configured with `BHIV_CORE_URL` (default `http://127.0.0.1:8003`).
- When forwarding is disabled the API response returns only the `blueprint` object. When enabled the response will include a `core_response` payload with the downstream Core response.
- Telemetry events are written to `db/creator_core_telemetry/insightflow_events.jsonl` for observability.

## Quick Integration Example

Add a new module in **under 1 minute**:

### Step 1: Create Module (30 seconds)
```python
# modules/calculator/module.py
from typing import Dict, Any

class CalculatorModule:
    def process(self, data: Dict[str, Any]) -> Dict[str, Any]:
        a = data.get("a", 0)
        b = data.get("b", 0)
        result = a + b
        return {
            "status": "success",
            "message": "Calculation completed",
            "result": {"sum": result}
        }
```

### Step 2: Update Models (15 seconds)
```python
# core/models.py - Add "calculator" to module field
module: Literal["finance", "education", "creator", "sample_text", "calculator"]
```

### Step 3: Register in Gateway (15 seconds)
```python
# core/gateway.py - Add import and registration
from modules.calculator.module import CalculatorModule

# In __init__:
"calculator": CalculatorModule()

# In process_request routing:
if module in ["sample_text", "calculator"]:
    response = agent.process(data)
```

**Done!** Test with: `POST /core` → `{"module": "calculator", "intent": "generate", "user_id": "test", "data": {"a": 5, "b": 3}}`

## Module Integration Guide

### Simple Module Structure

```
modules/
└── your_module/
    ├── __init__.py
    └── module.py
```

### Module Template

```python
from typing import Dict, Any

class YourModule:
    def process(self, data: Dict[str, Any]) -> Dict[str, Any]:
        # Your processing logic here
        return {
            "status": "success",
            "message": "Processing completed",
            "result": {"your_data": "result"}
        }
```

### Integration Steps

1. **Create module** in `modules/your_module/module.py`
2. **Add module name** to `core/models.py` Literal type
3. **Import and register** in `core/gateway.py`
4. **Route appropriately** (use `process()` for simple modules)

## API Endpoints

- `POST /core` - Main processing endpoint
- `GET /get-history?user_id=X` - User interaction history
- `GET /get-context?user_id=X` - Recent context (last 3 interactions)
- `GET /` - API information

## Memory Management

- **Automatic storage** of all interactions
- **5-entry limit** per user per module
- **SQLite database** for persistence (`db/context.db`)
